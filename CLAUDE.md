
● Clone 项目改进总结

  1. Bridge 网络 + 自动 IP 分配                                                                                                                           
  - 重写 auto_setup_network()，所有 VM 共享 clone-br0 bridge（172.30.0.0/16 网段）                                                                          - 顺序分配 IP（172.30.0.2、0.3...），支持 65534 个 VM
  - 用 ioctl 替代 ip 子进程创建 bridge、配置 IP，NAT 仍用 iptables                                                                                          - 文件: src/net/mod.rs                                                                                                                                                                                                                                                                                              2. 免 sudo 运行                                                                                                                                         
  - 通过 Linux capabilities（cap_net_admin, cap_net_raw）替代 sudo                                                                                          - 一次性 setcap 后所有 VM 操作无需 root
  - 文件: src/net/mod.rs, crates/clone-init/src/main.rs                                                                                                                                                                                                                                                               3. 自动持久化 Overlay                                                                                                                                   
  - --overlay 默认自动创建按 IP 命名的 overlay 文件（如 overlay-172.30.0.2.img）                                                                            - 宿主机上 mkfs.ext4 格式化，fallocate 预分配（避免稀疏文件导致块设备卡顿）
  - 支持 --overlay-size 参数（如 10G、20G），默认 4GB                                                                                                       - 文件: src/main.rs                                                                                                                                     
  4. 目录挂载（virtio-fs）                                                                                                                                
  - --shared-dir 支持三段格式 path:tag:mountpoint，自动挂载到 guest 指定路径                                                                                - 嵌入 virtiofs.ko 到 initrd，clone-init 自动加载和挂载
  - 文件: src/vmm/mod.rs, src/rootfs.rs, crates/clone-init/src/main.rs                                                                                                                                                                                                                                                5. Guest 自动配网

  - clone-init 读取内核参数 clone.net_ip/gw/mask，通过 ioctl 配置 eth0                                                                                      - 自动写 DNS（8.8.8.8）、添加默认路由
  - 文件: crates/clone-init/src/main.rs                                                                                                                                                                                                                                                                               6. SSH 登录优化                                                                                                                                         
  - clone-init 自动设置 UseDNS no 到 sshd_config，消除 DNS 反查延迟                                                                                         - 登录从数秒降到 0.1 秒
  - 文件: crates/clone-init/src/main.rs                                                                                                                                                                                                                                                                               7. VM 快照恢复（save/restore）                                                                                                                          
  - clone save: 暂停 VM → 保存内存 + vCPU + 设备状态 → 关机                                                                                                 - clone restore: 从快照恢复完整 VM 状态，内存中的进程继续运行
  - 保存 overlay 路径和 guest IP，restore 时自动恢复网络和磁盘                                                                                              - 文件: src/main.rs, src/vmm/mod.rs, src/boot/template.rs, src/control/sync_server.rs                                                                                                                                                                                                                               8. 稳定性修复                                                                                                                                           
  - fallocate 预分配替代稀疏文件，消除 blk_mq_requeue_work CPU 卡顿                                                                                         - 内核参数 blk_mq=0 禁用有问题的块设备多队列
  - clone-init 必须静态编译（RUSTFLAGS="-C target-feature=+crt-static"）                                                                                                                                                                                                                                              使用方式                                                                                                                                                
  # 启动 VM（自动配网 + 持久化 overlay + 目录挂载）                                                                                                         clone run --kernel vmlinuz --rootfs ubuntu-22.img --overlay --net \                                                                                         --shared-dir /host/path:tag:/guest/mount \                                                                                                                --overlay-size 10G --mem-mb 1024                                                                                                                                                                                                                                                                                  # 保存 VM
  clone save --output ./snapshots/my-vm
                                                                                                                                                            # 恢复 VM（进程继续运行）                                                                                                                                 clone restore --snapshot ./snapshots/my-vm --net
